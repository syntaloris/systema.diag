# SYSTEMA.diag

---

## 🚀 Быстрый старт

```bash
git clone https://github.com/syntaloris/systema.diag.git
cd systema.diag
# Открой systema_diag.html в браузере — всё работает офлайн
```

Или просто скачай `systema_diag.html` и открой в любом современном браузере (Chrome, Firefox, Edge, Safari).

## 📋 Расшифровка каждой строчки

### 🖥 Процессор & Нагрузка

| Параметр | Что означает | Откуда берётся |
|---|---|---|
| **Нагрузка JS-движка** | Процент загрузки JavaScript-движка в данный момент. Измеряется через математический бенчмарк (цикл вычислений). Показывает, насколько "занят" движок браузера, а не реальный CPU компьютера. | Бенчмарк в реальном времени |
| **Логических ядер** | Сколько процессорных ядер выделено браузеру для выполнения JavaScript. Например, 8 означает, что браузер может параллельно выполнять 8 потоков JS. | `navigator.hardwareConcurrency` |
| **Архитектура** | Архитектура процессора: x86 (Intel/AMD), ARM (Apple Silicon, мобильные) или unknown. Современные браузеры скрывают это через Client Hints API. | Client Hints API / User Agent |
| **Платформа (UA)** | Платформа из строки User Agent. Например: Win32 (Windows), MacIntel (macOS), Linux x86_64. | `navigator.platform` |
| **График нагрузки** | Живой график за последние 60 секунд. Показывает динамику загрузки JS-движка. | Canvas, обновляется каждую секунду |

> **Почему нет GHz и "класса устройства"?** Браузер никогда не сообщает реальную тактовую частоту процессора. Эвристика "посчитаем sqrt в цикле и выведем MHz" — это скорость исполнения кода в конкретном движке, а не GHz процессора. Мы не показываем то, что нельзя проверить.

### 📐 Viewport & Окно

| Параметр | Что означает | Откуда берётся |
|---|---|---|
| **window.innerWidth** | Ширина видимой области страницы **внутри** окна браузера. Не включает полосы прокрутки, рамки окна и панели инструментов. | `window.innerWidth` |
| **window.innerHeight** | Высота видимой области страницы внутри окна. | `window.innerHeight` |
| **window.outerWidth** | Ширина **всего** окна браузера — включая рамки, заголовок, панели вкладок и инструментов. Всегда ≥ innerWidth. | `window.outerWidth` |
| **window.outerHeight** | Высота **всего** окна браузера — включая заголовок и все панели. Всегда ≥ innerHeight. | `window.outerHeight` |
| **window.screenLeft / screenX** | Расстояние в пикселях от левого края **экрана** до левого края окна браузера. Показывает, где окно расположено на мониторе. | `window.screenLeft` или `screenX` |
| **window.screenTop / screenY** | Расстояние от верхнего края экрана до верхнего края окна браузера. | `window.screenTop` или `screenY` |
| **window.devicePixelRatio** | Соотношение физических пикселей экрана к CSS-пикселям. `1.0` = обычный экран. `2.0` = Retina / HiDPI (например, MacBook, современные смартфоны). `1.25` = Windows с масштабом 125%. | `window.devicePixelRatio` |
| **Visual Viewport** | Размер визуального viewport'а с учётом **масштаба** (pinch-zoom на телефоне или Ctrl+колесо). Может отличаться от innerWidth при зуме. | `window.visualViewport` |
| **Visual Scale** | Текущий масштаб страницы. `1.000` = 100% (без зума). `1.500` = увеличено в 1.5 раза. | `visualViewport.scale` |
| **Visual Offset** | Смещение визуального viewport'а относительно layout viewport при масштабировании. Показывает, насколько сдвинулась видимая область. | `visualViewport.pageLeft/Top` |
| **Aspect Ratio** | Соотношение сторон viewport'а (ширина ÷ высота). Например, 1.778 = 16:9. Также показывает класс: Wide, Ultrawide, Portrait и т.д. | Расчёт |

### 🖼 Экран (Screen Object)

| Параметр | Что означает | Откуда берётся |
|---|---|---|
| **screen.width** | Физическое разрешение экрана по ширине в пикселях. Это **весь монитор**, не только окно браузера. Например, 1920 для Full HD. | `screen.width` |
| **screen.height** | Физическое разрешение экрана по высоте. Например, 1080 для Full HD. | `screen.height` |
| **screen.availWidth** | Доступная ширина экрана за вычетом системных панелей (панель задач Windows, Dock macOS, панели Linux). | `screen.availWidth` |
| **screen.availHeight** | Доступная высота экрана за вычетом системных панелей. | `screen.availHeight` |
| **screen.availLeft** | Отступ виртуального экрана слева. Если > 0, возможно, монитор расположен справа от основного в многомониторной установке. | `screen.availLeft` |
| **screen.availTop** | Отступ виртуального экрана сверху. Если > 0, монитор расположен ниже основного. | `screen.availTop` |
| **screen.colorDepth** | Глубина цвета — сколько бит используется для представления цвета одного пикселя. Обычно 24 (8 бит на каждый канал R, G, B). | `screen.colorDepth` |
| **screen.pixelDepth** | Глубина пикселя. На практике всегда совпадает с colorDepth (24 бита). | `screen.pixelDepth` |
| **Ориентация** | Текущая ориентация экрана: `landscape-primary` (альбомная, обычная), `portrait-primary` (портретная, как телефон), `landscape-secondary` (перевёрнутая). | `screen.orientation.type` |
| **Угол поворота** | Угол поворота экрана относительно естественной ориентации устройства: 0° (обычно), 90° (повёрнут), 180° (перевёрнут), 270°. | `screen.orientation.angle` |

> **Важно:** `screen.width/height` — это **физическое разрешение монитора**. `window.innerWidth/Height` — это **размер окна браузера**. Это разные вещи. Если у вас 4K-монитор (3840×2160), но браузер в окне на половину экрана, то screen.width = 3840, а window.innerWidth ≈ 1920.

### 🎨 Цвет & Медиа (CSS Media Queries)

| Параметр | Что означает | Откуда берётся |
|---|---|---|
| **Color Scheme** | Предпочтение пользователя: **Dark** (тёмная тема) или **Light** (светлая). Браузер/система передаёт это сайтам, чтобы они автоматически переключались. | `prefers-color-scheme` |
| **Reduced Motion** | Пользователь включил режим уменьшения анимаций (в Windows/macOS/iOS). Важно для людей с вестибулярными нарушениями — сайт должен убрать параллакс и быстрые переходы. | `prefers-reduced-motion` |
| **High Contrast** | Включён ли режим повышенной контрастности в системе. Используется людьми с нарушениями зрения. | `prefers-contrast` |
| **Forced Colors** | Принудительная цветовая схема системы (например, Windows High Contrast). Сайт должен адаптироваться под системные цвета. | `forced-colors` |
| **Inverted Colors** | Инвертированы ли цвета экрана (негатив). Редко используется, но важно для доступности. | `inverted-colors` |
| **Monochrome** | Монохромный режим — оттенки серого. Используется на e-ink экранах или для экономии зрения. | `monochrome` |
| **HDR / Dynamic Range** | Поддерживает ли экран HDR-контент (High Dynamic Range — широкий динамический диапазон яркости). Важно для видео и игр. | `dynamic-range` |
| **Display Mode** | Режим отображения: **Browser** (обычный браузер), **Standalone** (как отдельное приложение/PWA), **Fullscreen** (полноэкранный). | `display-mode` |
| **Pointer / Hover** | Тип указателя: **fine** = мышь/тачпад (точный), **coarse** = тачскрин (палец, неточный). **hover** = поддерживается наведение, **none** = только тап. | `any-pointer` / `any-hover` |

> **Почему нет PPI?** Браузер не знает физический размер вашего монитора в дюймах. Формула `DPR × 96` — это CSS-договорённость, а не реальная плотность пикселей. 96 — это условное "стандартное" значение, не измеренное.

### 🎮 GPU Рендерер

| Параметр | Что означает | Откуда берётся |
|---|---|---|
| **Unmasked Renderer** | Реальное название вашей видеокарты. Например: "NVIDIA GeForce RTX 3060", "Apple M1 GPU", "Intel Iris Xe". | `WEBGL_debug_renderer_info` |
| **Unmasked Vendor** | Реальный производитель видеокарты: NVIDIA Corporation, Apple Inc., Intel Inc. и т.д. | `WEBGL_debug_renderer_info` |
| **WebGL Version** | Версия графической библиотеки WebGL в браузере. Например: "WebGL 2.0 (OpenGL ES 3.0 Chromium)". | `gl.getParameter(VERSION)` |
| **GLSL Version** | Версия языка шейдеров GLSL (GL Shading Language). Например: "WebGL GLSL ES 3.00". | `gl.getParameter(SHADING_LANGUAGE_VERSION)` |
| **Vendor (masked)** | Маскированное имя производителя (без расширения приватности). Обычно показывает "WebKit" или "Mozilla" вместо реального вендора. | `gl.getParameter(VENDOR)` |
| **Renderer (masked)** | Маскированное имя видеокарты. Обычно generic строка вроде "WebKit WebGL" вместо реальной модели GPU. | `gl.getParameter(RENDERER)` |
| **Hardware Accel** | Включено ли аппаратное ускорение графики. Если "Нет" — всё рисуется процессором (медленно). | Feature detection |
| **WebGL Context** | Версия WebGL-контекста: **WebGL 2.0** (современный) или **WebGL 1.0** (устаревший). | `canvas.getContext('webgl2')` |
| **Extensions Count** | Количество доступных расширений WebGL — дополнительных возможностей графики. | `gl.getSupportedExtensions().length` |

> **Важно:** В некоторых браузерах (Firefox в приватном режиме, Safari) расширение `WEBGL_debug_renderer_info` заблокировано для защиты от фингерпринтинга. Тогда вместо "NVIDIA GeForce" будет написано "Заблокировано (приватность)".

### 📐 WebGL Limits

| Параметр | Что означает | Зачем нужно |
|---|---|---|
| **Max Texture Size** | Максимальный размер одной 2D-текстуры (ширина и высота). Например, 16384 означает, что можно загрузить изображение 16384×16384 пикселей. | Для игр и 3D — ограничение на размер текстур |
| **Max Viewport** | Максимальные размеры viewport'а для рендеринга. Обычно совпадает с Max Texture Size. | Размер буфера, в который рисует GPU |
| **Max Cube Map** | Максимальный размер текстуры кубической карты (6 граней для skybox/отражений). | 3D-графика, отражения окружения |
| **Max Renderbuffer** | Максимальный размер renderbuffer — специального буфера для промежуточного рендеринга (тени, пост-эффекты). | Постобработка, теневые карты |
| **Max Anisotropy** | Максимальная степень анизотропной фильтрации текстур. Чем выше — тем чётче текстуры под углом. Качество vs производительность. | Фильтрация текстур в играх |
| **Max Draw Buffers** | Максимальное количество одновременных буферов рендеринга (MRT — Multiple Render Targets). WebGL 2.0 позволяет рисовать сразу в несколько текстур. | Deferred shading, G-buffer |
| **Max Color Attachments** | Максимум цветовых вложений (сколько цветовых буферов можно прикрепить к фреймбуферу). | Сложный рендеринг |
| **Max Samples (MSAA)** | Максимальное количество сэмплов для мультисэмплинга (MSAA — сглаживание краёв). Например, 4 или 8. | Антиалиасинг, сглаживание лесенок |
| **Max 3D Texture** | Максимальный размер 3D-текстуры (текстура с глубиной). Только для WebGL 2.0. | Объёмный рендеринг, медицинская визуализация |

### 🔢 WebGL Shaders

| Параметр | Что означает | Зачем нужно |
|---|---|---|
| **Max Vertex Attribs** | Максимальное количество vertex attributes — входных данных для каждой вершины (координаты, цвет, нормаль, UV). Обычно 16. | 3D-модели — сколько данных можно передать на вершину |
| **Max Vertex Uniforms** | Максимум uniform-переменных в вершинном шейдере (константы: матрицы, цвета, настройки). | Сложность вершинного шейдера |
| **Max Fragment Uniforms** | Максимум uniform-переменных в фрагментном (пиксельном) шейдере. | Сложность освещения, материалов |
| **Max Varying Vectors** | Максимум varying-векторов — данных, передаваемых от вершинного шейдера к пиксельному (цвет, нормаль, UV). | Качество интерполяции между вершинами |
| **Max Tex Units (FS)** | Максимум текстурных юнитов (слотов) во фрагментном шейдере. Сколько текстур можно использовать одновременно в материале. | Многослойные материалы |
| **Max Tex Units (VS)** | Максимум текстурных юнитов в вершинном шейдере. | Дисплейсмент, вершинные текстуры |
| **Max Combined Tex** | Общий максимум текстурных юнитов для всех шейдеров вместе. | Лимит текстур на весь draw call |
| **Aliased Point Size** | Диапазон размеров точек (point sprites) — от минимума до максимума в пикселях. | Частицы, спрайты |
| **Aliased Line Width** | Диапазон ширины линий без сглаживания. Обычно 1–1 (только 1 пиксель). | Wireframe, линии |

### 💻 ОС & Платформа

| Параметр | Что означает |
|---|---|
| **ОС (detected)** | Автоматически определённая операционная система по анализу строки User Agent. Например: Windows 10/11, macOS, Android 14, iOS 17, Linux. |
| **ОС (UserAgentData)** | ОС из современного Client Hints API — более надёжный способ, чем парсинг строки. Не все браузеры поддерживают. |
| **Платформа** | Платформа из `navigator.platform`. Например: Win32, MacIntel, Linux x86_64. |
| **User Agent** | Полная строка идентификации браузера. Содержит ОС, версию браузера, движок. Например: "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36..." |
| **App Version** | Версия приложения (браузера). Обычно начинается с "5.0" из исторических причин. |
| **App CodeName** | Кодовое имя приложения. Практически всегда "Mozilla" — реликт истории браузерных войн. |
| **Product** | Название продукта (движка). Практически всегда "Gecko". |
| **Product Sub** | Дополнительная информация о продукте. Обычно версия движка, например "20100101". |
| **Vendor** | Производитель браузера. Например: "Google Inc.", "Apple Computer, Inc.", "" (Firefox). |
| **Build ID** | Уникальный идентификатор сборки браузера. Только в Firefox. |
| **oscpu** | Архитектура CPU из Firefox. Например: "Windows NT 10.0; Win64; x64". |

### 🔧 Устройство & Железо

| Параметр | Что означает |
|---|---|
| **CPU Cores** | Количество логических процессорных ядер, доступных браузеру. Не путать с физическими ядрами — это именно то, что видит JS. |
| **Max Touch Points** | Сколько пальцев одновременно может распознать сенсорный экран. 0 = нет тача, 5 = стандарт для смартфонов, 10 = премиум. |
| **Touch Support** | Есть ли у устройства сенсорный ввод. "Да (5)" означает 5 точек касания. |
| **Keyboard API** | Доступен ли API для работы с физической клавиатурой (раскладка, нажатия). |
| **Virtual Keyboard** | Доступен ли API виртуальной клавиатуры (для управления экранной клавиатурой на мобильных/планшетах). |
| **PDF Viewer** | Встроен ли в браузер просмотрщик PDF или используется внешняя программа (Adobe, системный viewer). |
| **WebDriver** | Обнаружен ли WebDriver — признак автоматизированного управления (Selenium, Puppeteer, Playwright). Если "Да" — возможно, это бот или тест. |
| **Standalone (PWA)** | Запущен ли сайт как отдельное приложение (Progressive Web App) без адресной строки, или в обычном браузере. |
| **Mobile** | Определено ли устройство как мобильное: нет hover + coarse pointer (тачскрин). |
| **Tablet** | Определено ли устройство как планшет: нет hover + coarse pointer + ширина экрана ≥ 600px. |

> **Почему нет RAM?** Браузерный API `navigator.deviceMemory` возвращает **приблизительное** значение с жёстким потолком (обычно максимум 8 ГБ). У вас может быть 32 ГБ оперативной памяти, а браузер скажет "8" — это ограничение самого API Chrome, а не бага. Мы не показываем данные, которые заведомо врут.

### 🕒 Время & Локаль

| Параметр | Что означает |
|---|---|
| **Timezone** | Ваш часовой пояс по стандарту IANA. Например: `Europe/Moscow`, `America/New_York`, `Asia/Tokyo`. |
| **Timezone Offset** | Разница между вашим локальным временем и UTC (Всемирное координированное время) в часах. Москва: +3h, Нью-Йорк: -4h (летом). |
| **Locale Date** | Текущая дата в формате, принятом в вашей локали. Например: "23.05.2026" (Россия) или "5/23/2026" (США). |
| **ISO String** | Дата и время в международном формате ISO 8601. Например: `2026-05-23T21:43:00.000Z`. Стандарт для обмена данными. |
| **Performance.now()** | Время в миллисекундах с момента открытия страницы. Высокоточный таймер (микросекундная точность). Не связан с системным временем. |
| **Epoch Time** | Unix-время — количество миллисекунд с 1 января 1970 года, 00:00:00 UTC. Глобальный стандарт для меток времени. |
| **Intl.DateTimeFormat** | Поддержка API для форматирования дат под разные страны (названия месяцев, порядок день/месяц/год). |
| **Intl.NumberFormat** | Поддержка API для форматирования чисел (разделители: точка/запятая, валюты, проценты). |
| **Intl.Collator** | Поддержка API для правильного сравнения и сортировки строк с учётом языка (например, буква "Ё" в русском). |
| **Intl.PluralRules** | Поддержка API для правил множественного числа в разных языках (1 яблоко, 2 яблока, 5 яблок). |
| **Intl.RelativeTime** | Поддержка API для относительного времени: "2 часа назад", "через 3 дня". |

### 🌐 Сеть

| Параметр | Что означает |
|---|---|
| **Effective Type** | Эффективный тип сети по мнению браузера: **4g** (быстрая), **3g** (средняя), **2g** (медленная), **slow-2g** (очень медленная). Используется сайтами для адаптации контента. |
| **Downlink** | Приблизительная скорость скачивания в мегабитах в секунду (Mbps) по данным браузера/операционной системы. |
| **Downlink Max** | Максимальная заявленная скорость соединения (теоретический потолок). |
| **RTT (Ping)** | Round-trip time — время отклика до ближайшего сетевого узла в миллисекундах. Меньше = лучше (игры, видеозвонки). |
| **Save Data** | Включён ли режим экономии трафика (Data Saver). Если "Включён" — сайты должны отдавать облегчённый контент. |
| **Connection Type** | Тип подключения: **wifi**, **cellular** (мобильная сеть), **ethernet** (провод), **none**. |
| **Онлайн** | Есть ли у устройства подключение к интернету прямо сейчас. Не гарантирует доступность конкретного сайта. |
| **Network Info API** | Доступен ли в браузере Network Information API. Не все браузеры поддерживают (Firefox не поддерживает). |

### 📡 URL & Протокол

| Параметр | Что означает |
|---|---|
| **Протокол** | Протокол текущей страницы: `https:` (защищённый, шифрование) или `http:` (нешифрованный). |

| **Referrer** | Адрес страницы, с которой пользователь перешёл на текущую. Может быть скрыт политикой приватности. |
| **CSP** | Content Security Policy — защитная политика, ограничивающая источники скриптов, стилей, картинок. Предотвращает XSS-атаки. |
| **Feature Policy** | Устаревшее API управления доступом к функциям браузера (камера, геолокация, микрофон). Заменено на Permissions Policy. |
| **Permissions Policy** | Современное API управления разрешениями. Определяет, какие функции доступны странице и iframe'ам. |

### ⏱ Performance Timing

| Параметр | Что означает |
|---|---|
| **TTFB** | Time To First Byte — время от отправки запроса до получения **первого байта** ответа от сервера. Ключевой метрик скорости сервера. |
| **DNS Lookup** | Время DNS-разрешения — преобразование доменного имени (example.com) в IP-адрес сервера. |
| **TCP Connect** | Время установки TCP-соединения (трёхэтапное рукопожатие). |
| **SSL Handshake** | Время TLS/SSL-рукопожатия — установка защищённого шифрованного соединения (HTTPS). |
| **Response Time** | Время загрузки тела ответа от сервера (весь HTML-документ). |
| **DOM Processing** | Время обработки и построения DOM-дерева страницы (парсинг HTML, выполнение скриптов). |
| **Load Event** | Время до события `load` — страница полностью загружена включая картинки, стили, шрифты. |
| **Total Resources** | Количество всех загруженных ресурсов: HTML, CSS, JS, картинки, шрифты, API-запросы. |
| **Navigation Type** | Тип перехода: **navigate** (обычный переход по ссылке), **reload** (F5/Ctrl+R), **back_forward** (кнопки назад/вперёд). |

> **First Paint / FCP / LCP** — дополнительные метрики, выводятся в системный лог: First Paint (первый пиксель), FCP (First Contentful Paint — первый значимый контент), LCP (Largest Contentful Paint — самый большой элемент).

|---|---|
| **Secure Context** | Безопасный контекст — страница загружена по HTTPS, с localhost или с локального файла. Некоторые API (геймпад, clipboard, WebAuthn) работают только в secure context. |

### 📦 Хранилища (доступность API)

| Параметр | Что означает |
|---|---|
| **LocalStorage** | Локальное хранилище данных в браузере. Данные сохраняются навсегда (до очистки). Лимит ~5-10 МБ. Используется для настроек, токенов. |
| **SessionStorage** | Временное хранилище. Данные живут только пока открыта вкладка. При закрытии — удаляются. |
| **IndexedDB** | База данных в браузере. Может хранить сотни мегабайт (до квоты). Используется для офлайн-приложений, кэшей, документов. |
| **Cache API** | API для кэширования ресурсов. Используется Service Workers для офлайн-работы (PWA). |
| **OPFS** | Origin Private File System — приватная файловая система, доступная только origin'у (вашему домену). Для хранения больших файлов. |
| **Origin Private FS** | Альтернативная проверка доступности файловой системы через File System Access API. |

> **Почему нет Quota / Usage / Persisted?** `navigator.storage.estimate()` даёт приблизительные цифры, которые сильно варьируются между браузерами и не всегда доступны. Мы показываем только факт доступности API, а не гадание о гигабайтах.

| Параметр | Что означает |
|---|---|
| **AudioContext** | Web Audio API — доступность аудиоконтекста для обработки звука: фильтры, эквалайзер, 3D-звук, анализатор спектра. |

## ❌ Что НЕ показывает (и почему)

| Параметр | Почему скрыто |
|---|---|
| **Общий объём RAM** | `navigator.deviceMemory` возвращает приблизительное значение с жёстким потолком (обычно макс. 8 ГБ). У вас 32 ГБ — браузер скажет "8". Это ограничение API, а не баг. |
| **Использование RAM** | `performance.memory` показывает только JS Heap (Chrome-only), а не системную RAM. |
| **Тактовая частота CPU** | Браузер не предоставляет MHz. Эвристики через бенчмарк — это скорость JS, не GHz процессора. |
| **Класс устройства** | "High-end" по количеству ядер — спекуляция. 4 ядра на телефоне ≠ 4 ядра на десктопе. |
| **Батарея** | Battery API deprecated и удалён в Firefox/Safari. Данные ненадёжны. |
| **PPI / DPI экрана** | Браузер не знает физический размер монитора в дюймах. `DPR × 96` — CSS-договорённость, не измерение. |
| **Количество мониторов** | Эвристика по `availLeft` ненадёжна. Нет точного API. |
| **Локальный IP** | WebRTC ICE — блокируется VPN, выдаёт рандомные IP (MDNS), устаревает как метод. |
| **Storage Quota / Usage** | `storage.estimate()` — приблизительный, варируется между браузерами. |
| **Audio Latency** | `baseLatency` часто возвращает `0`, зависит от аудиодрайвера. |
| **GPS-координаты** | Требует явного разрешения пользователя. Без разрешения — только факт доступности API. |
| **Температура CPU / вентиляторы** | Нативный доступ запрещён песочницей браузера по соображениям безопасности. |

## 🛡 Приватность

SYSTEMA.diag **не отправляет данные никуда**. Всё выполняется локально в вашем браузере. Никаких запросов на сервер, никакой телеметрии, никаких cookies, никакого fingerprinting.

Используется только стандартный Web API (`navigator`, `window`, `screen`, `performance`, `WebGL`).
